# レビュー手順

## 1. 用語の定義確認

レビュー対象内で使われている重要な用語について、定義が明示されているか確認する。

- 定義されていない場合：どの意味で使われているか特定し、注釈として定義を提案する。
- 複数の解釈が可能な場合：それぞれの解釈を列挙し、本スコープでの意味を明確にするよう求める。

## 2. 論理構成の検証

以下の観点で論理の整合性を確認する。

- 前提（公理・定義）が明示されているか
- 各主張が前提から正しく導出されているか（飛躍がないか）
- 暗黙の仮定が含まれていないか。含まれている場合は明示するよう指摘する。
- 矛盾する記述がないか

## 3. 出典の検証

- 主張に出典が付いているか確認する。
- 出典がある場合、一次ソース（公式ドキュメント、RFC、言語仕様、論文など）であるか確認する。
- 二次ソースのみの場合、一次ソースへの差し替えまたは併記を提案する。
- 出典の内容と主張が整合しているか確認する。

## 4. レビューコメントの形式

レビューコメント自体も以下を守る。

- 指摘の根拠を示す。「こうした方がいい」だけでなく「なぜなら〜だから」を含める。
- 参照した一次ソースのURLまたは文献名を付記する。
- 推測を含む場合は明示する。
