# スコープチェック手順

## 1. 現状の整理

以下をまとめて出力する。

- 当初合意したTODOリスト
- 各TODOの状態（完了 / 進行中 / 未着手）
- 会話の中で新たに浮上した論点・疑問

## 2. 逸脱の判定

現在の議論が当初のTODOのいずれかに該当するか判定する。

- 該当する場合：そのまま続行。
- 該当しない場合：新しい論点として明示し、以下を確認する。
  - この新論点は、当初の問題より本質的か？
  - 当初のTODOに戻るべきか、それともこちらを優先すべきか？

## 3. セッション分割の提案

新論点を別セッションで扱うべきと判断した場合、以下の形式でコンテキストをまとめる。

```
## 新セッション用コンテキスト

### 背景
（なぜこの論点が浮上したか）

### 解決したいこと
（具体的な問い・ゴール）

### 現時点での仮説・わかっていること
（ここまでの議論で得られた情報）

### 関連ファイル・リソース
（参照すべきファイルやURL）
```

このコンテキストを出力した上で、ユーザーに「このまま続けるか、新セッションに切り出すか」を確認する。
